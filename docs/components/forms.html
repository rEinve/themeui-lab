Form Guide

Start here when you need to build or document a form in themeui-lab. This guide pulls together the control docs, shared field language, accessibility contract, and layout boundary in one place.

Forms span two layers: primitives control page layout and grouping, while components control the inside of each field or choice control. Keep section spacing, container width, and grid decisions outside the form components themselves.

1. Start with field messaging

Use the shared label, hint, success, warning, and error helpers before choosing the specific control.

Open Field Messaging

2. Choose the right control

Use input or textarea for text entry, select for bounded options, and radio, checkbox, or switch for choices.

Open Components Directory

3. Wire accessibility

Every field needs a programmatic label, connected help text, and linked validation feedback when present.

Open Form Example

4. Keep layout in primitives

Use stack, grid, section, and container to compose the page shell around the form rather than pushing layout into controls.

Accessibility contract

This is the baseline contract the form example follows.

Every input should have a programmatic label. Prefer explicit label for plus control id, connect helper or validation copy with aria-describedby, and pair invalid states with aria-invalid="true" and an error message when available.
