Components

Core components are reusable UI pieces that style their own interior and state only.

Canonical component classes are .button, .card, .input, .textarea, .tag, and .divider. The legacy .btn alias remains temporarily for compatibility only.

Surfaces

Card and divider support content presentation.

Button + Tag

Components should stay small and reusable, then compose together inside flow primitives.

Draft 
<div class="cluster">
  <button class="button" type="button">Save Changes</button>
  <span class="tag">Draft</span>
</div>

Card with Form Controls

Cards provide a neutral surface. Inputs and textareas keep their own interior styling.

Profile

<article class="card stack stack--sm">
  <h3 class="card__title">Profile</h3>
  <input class="input" type="text" placeholder="Display name" />
  <textarea class="textarea" placeholder="Short bio"></textarea>
</article>
