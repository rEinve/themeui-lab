Foundations Examples

Foundations define the token contract for color, spacing, typography, radius, shadows, and motion.

Canonical consumption should prefer exported CSS variables like --color-*, --space-*, --radius-*, --shadow-*, and --motion-*. Legacy aliases still exist for compatibility, but new work should use the canonical token names.

Radius Tokens

Radius tokens are grouped by intent so surfaces, media, and controls keep a consistent shape language.

Prefer semantic radius tokens in product code: --radius-container-* for cards and panels, --radius-media-* for images and media frames, and --radius-control-* for buttons and form fields. The base scale --radius-s through --radius-full is still available for utilities and one-off composition helpers.

Base Scale

--radius-s: 0.5rem

--radius-m: 0.875rem

--radius-l: 1.5rem

--radius-xl: 2rem

--radius-full: 9999px

Semantic Families

--radius-container-sm: 1rem

--radius-container-md: 1.5rem

--radius-container-lg: 2rem

--radius-media-sm: 0.75rem

--radius-media-md: 1rem

--radius-media-lg: 1.5rem

--radius-control-sm: 0.625rem

--radius-control-md: 0.875rem

--radius-control-lg: 1rem

--radius-pill: 999px

Do

Pick the family that matches the UI role so buttons, cards, and media stay visually consistent across the system.

Don't

Mix container and control radii arbitrarily inside the same pattern unless the shape contrast is an intentional design choice.

Container Surface

Cards, panels, and grouped surfaces should usually start with --radius-container-md.

pill tag 
.card {
  border-radius: var(--radius-container-md);
}

.button,
.input,
.textarea,
.select {
  border-radius: var(--radius-control-md);
}

.tag {
  border-radius: var(--radius-pill);
}
