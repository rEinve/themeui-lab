Patterns

Patterns combine primitives and components into larger page-level structures. They are not core UI components.

Canonical pattern usage should stay in the pattern layer. Top bar and off-canvas navigation belong here because they manage page shell, responsive behavior, and navigation structure. Hero belongs here too because it composes page regions, content flow, and components into one landing-page section.

Hero

Use the hero pattern for a landing-page introduction with headline, supporting copy, actions, and an optional visual area.

Landing Region

Use card, button, tag, and layout primitives inside the hero instead of inventing hero-only UI elements.

<section class="hero">
  <div class="hero__content">
    <span class="tag">themeui-lab</span>
    <h1>Minimal hero pattern</h1>
    <p>Use hero for landing-page introduction, not as a generic card replacement.</p>
    <div class="hero__actions">
      <a class="button" href="/demo/hero.html">Open Demo</a>
    </div>
  </div>
  <div class="hero__media">
    <article class="card">...</article>
  </div>
</section>

App Shell

Use the existing app shell pattern for a responsive top bar that becomes an off-canvas drawer on smaller screens.

Pattern classes:

.header .nav-container .site-nav .nav-toggle .nav-drawer .nav-links data-nav-open data-nav-drawer 
<header class="header">
  <div class="container nav-container">
    <a class="navbar-brand" href="/">themeui-lab</a>
    <nav class="site-nav" data-app-nav>
      <button class="nav-toggle" type="button" aria-controls="demo-nav-drawer" data-nav-open>Menu</button>
      <div class="nav-overlay" data-nav-overlay></div>
      <div class="nav-drawer" id="demo-nav-drawer" data-nav-drawer>...</div>
    </nav>
  </div>
</header>
