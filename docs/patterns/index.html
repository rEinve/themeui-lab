Patterns

Patterns compose primitives and components into larger page sections and application structures.

Use patterns when a page needs coordinated structure across multiple regions, behaviors, and reusable components. Patterns are not base UI elements and should not replace components.

Page Assemblies

Patterns are the right place for app shells, editorial rails, landing heroes, and other multi-region structures.

Behavior + Layout

A pattern can coordinate layout, navigation behavior, and component placement when those pieces belong to one page section.

Reuse Components

Patterns should be built from existing primitives and components instead of introducing pattern-only versions of basic UI.

Boundary Rule

Keep page composition in patterns, but keep portable surfaces and controls in the component layer.

Pattern Directory

These pages document the larger pattern assemblies that sit above components in the system.
