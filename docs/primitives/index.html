Primitives

Primitives handle page shell and content flow layout: structure, rhythm, and grouping.

Canonical usage: .container, .section, .grid-12, .stack, .row, .cluster. Alignment helpers like .items-center, .justify-between, and .text-center live in utilities and should only refine an existing layout primitive. Legacy aliases like .frame, .section-shell, .page-grid, .content-grid, .stack-tight, and .stack-loose remain for compatibility only.

Alignment Utilities

Use alignment helpers to refine an existing primitive, not to replace the primitive itself.

Contract: start with .row, .cluster, .stack, or .grid-12, then add a utility only for the alignment adjustment you need. Use .items-* for cross-axis alignment, .justify-* for main-axis distribution, and .self-* when one child needs different alignment from its siblings. Use .place-center only for simple two-axis centering on grid containers, and .text-* for inline text alignment.

Do

Combine a layout primitive with one or two alignment helpers when a specific section or one child needs different positioning.

Don't

Invent one-off component classes just to center, spread, or individually offset content that already belongs to a primitive.
