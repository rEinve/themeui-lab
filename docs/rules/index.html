System Rules

These rules keep the design system small, reusable, and resistant to layer drift.

The most important boundary is simple: primitives control layout and components control interior structure. Everything else supports that separation.

Use Tokens First

Prefer shared tokens and CSS variables over hard-coded values so visual changes stay centralized.

Keep Roles Clear

Use utilities for small adjustments, primitives for layout, components for reusable UI, and patterns for page-level assemblies.

Prefer Composition

Build with existing pieces before introducing new abstractions or special-case selectors.

Keep the System Small

Add only stable, reusable building blocks that improve the shared system rather than one product surface.
