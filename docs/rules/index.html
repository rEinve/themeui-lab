System Rules

Use these principles to keep the design system consistent, reusable, and resistant to design drift.

Primitives Control Layout

Use primitives like section, container, grid, stack, row, and cluster for structure and rhythm.

Components Control Interior Structure

Components style their own surface and internal content. They should not own page spacing or page shell layout.

Use a Three-Layer Page Structure

Page shell controls structure. Content flow controls rhythm. Components control their own interior.

Use Tokens, Not Hard-Coded Values

Prefer design tokens and exported CSS variables for color, spacing, typography, radius, motion, and shadows.

Prefer Composition Over Customization

Compose with existing primitives and components before creating special-case styles or one-off overrides.

Keep the System Small

Add only stable, reusable building blocks. Leave product-specific needs outside core system layers.
